── USER EXPERIENCE IMPROVEMENTS ─────────────────────────────────────

MISSING: Progress indicator on long blog posts
  Add a reading progress bar at the top of each article.
  Improves engagement metrics (dwell time) which Google uses as a
  ranking signal.

MISSING: Related posts widget based on category
  The current related posts are manually specified. Consider a
  tag-based automatic system as content grows.

MISSING: "Back to top" button on long pages
  Simple quality-of-life improvement that reduces bounce rate on mobile.

MISSING: Breadcrumb navigation on blog posts
  Home > Blog > Category > Article Title
  Improves both UX and SEO (Google shows breadcrumbs in search results).

MISSING: Table of Contents visibility on mobile
  TOC is currently visible but collapses awkwardly on small screens.
  Add a collapsible "Table of Contents" toggle for mobile.

MISSING: Search functionality across blog posts
  As content grows to 20+ posts, readers need to find specific topics.
  Add a simple search bar to blog.html.

── SEO TECHNICAL IMPROVEMENTS ───────────────────────────────────────

PRIORITY 1 — Canonical tags
  Add <link rel="canonical" href="https://aivoraai.com/page.html" />
  to every page. Prevents duplicate content issues as site grows.

PRIORITY 2 — Open Graph images
  Add <meta property="og:image"> tags with actual images for each page.
  Without these, social shares show a blank box — dramatically lower CTR.
  Minimum: Create one branded OG image (1200x630px) for the homepage,
  and unique images for each blog post category.

PRIORITY 3 — Structured data (Schema.org)
  Add Article schema to each blog post:
    @type: BlogPosting
    headline, datePublished, dateModified, author, description
  Add Organization schema to homepage.
  Add BreadcrumbList schema to post pages.
  Google may show rich results (star ratings, dates) which improves CTR by 20-30%.

PRIORITY 4 — Page speed optimisation
  Current issue: Google Fonts loaded via external CDN adds ~150-200ms
  latency. Solution: Preload font files or use font-display: swap.
  Target: Core Web Vitals all green (LCP < 2.5s, CLS < 0.1, FID < 100ms)

PRIORITY 5 — Internal linking audit
  Run a crawl (Screaming Frog free tier) monthly to check:
    - All blog posts have at least 2-3 internal links
    - No orphan pages (pages with zero internal links pointing to them)
    - No broken internal links

── CONVERSION OPTIMISATION ──────────────────────────────────────────

MISSING: Exit-intent popup
  When a reader moves their cursor toward the browser close button,
  show a newsletter signup: "Before you go — get our free weekly AI digest!"
  Expected impact: 2-5% of exiting visitors will convert to subscribers.

MISSING: Comparison table for top 3 tools in each niche
  Readers who reach a product page want to compare before deciding.
  Add a sticky comparison widget to product.html (already has sidebar filters,
  needs a "Compare" feature for selected tools).

MISSING: Social proof numbers on homepage
  "50,000+ users" and "200+ tools" are mentioned in stats but not
  prominently displayed on the hero section. Make these visible above the fold.

IMPROVEMENT: Footer affiliate disclosure
  Currently reads "Some links are affiliate links". Strengthen to:
  "This site contains affiliate links. We may earn a commission when
  you purchase through our links — at no additional cost to you.
  All recommendations are independent and based on testing."

Each new blog post was designed to avoid the following common patterns
found in AI affiliate and tools content:

  ✅ Decision frameworks for niche/tool/stack selection
  ✅ Myth-busting sections (Blog 5)
  ✅ Step-numbered processes with action verbs
  ✅ Internal cross-referencing builds topical authority cluster
